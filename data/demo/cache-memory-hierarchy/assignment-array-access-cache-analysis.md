# 과제 요약 - 배열 접근 패턴과 Cache 성능

## 실험 목적

배열 접근 패턴에 따라 cache miss와 실행 시간이 어떻게 달라지는지 확인한다.

## 비교한 접근 방식

1. 순차 접근
2. 큰 간격으로 건너뛰는 접근

## 관찰 결과

- 순차 접근은 인접한 메모리 주소를 연속적으로 읽기 때문에 spatial locality가 높았다.
- 큰 간격으로 건너뛰는 접근은 locality가 낮아 cache miss가 더 자주 발생했다.
- 같은 크기의 배열이라도 접근 방식에 따라 실행 시간 차이가 발생했다.

## 해석

강의에서 배운 cache memory와 locality 개념으로 결과를 설명할 수 있다.
순차 접근은 cache block에 포함된 nearby data를 다시 활용할 가능성이 높다.
반면 비순차 접근은 가져온 block을 충분히 활용하지 못해 miss 비용이 증가할 수 있다.

## 결론

프로그램 성능은 알고리즘뿐 아니라 메모리 접근 패턴의 영향을 크게 받는다.
cache friendly한 접근은 평균 메모리 접근 시간을 낮추는 데 유리하다.
